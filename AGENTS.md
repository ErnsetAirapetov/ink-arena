# AGENTS.md — InkArena

InkArena — веб-игра про рисование заклинаний. Кодовая база вехи 1 — monorepo
на npm workspaces: `packages/core` (детерминированное ядро без DOM) и
`packages/client` (веб-клиент на Vite + Canvas). Тех-дизайн — в
`docs/tech/architecture.md`. Параллельно идёт GDD в `docs/game/`, кухня
проработки — `docs/design/` (см. `docs/README.md`). Прототип снесён начисто
(Р43) и сохранён тегом `prototype-archive`.

## Рабочий процесс: оркестратор и задачи
- В репозитории работает одна сессия-оркестратор (модель Fable): она
  декомпозирует работу, ставит задачи и принимает результат. Исполнители —
  отдельные субагенты на Opus (сложные задачи) и Sonnet (рутинные).
- Доска задач — GitHub Issues репозитория, доступ через `gh` CLI напрямую
  (не через MCP — меньше токенов).
- Если ты субагент-исполнитель: тебе дан номер issue — работай в его рамках,
  по завершении верни итог оркестратору; issue закрывает оркестратор.
- Задачи, затрагивающие код, не выполняются мимо доски.

## Git-процесс для исполнителя
- Главная ветка — `main`, напрямую в неё не коммитим.
- Изолированный worktree ты получаешь автоматически (isolation:"worktree",
  каталог `.claude/worktrees/agent-<id>`). Создай в нём рабочую ветку:
  `git switch -c <ветка>`. Вручную `git worktree add` не делай. Чужие
  worktree не трогай — параллельно работают другие агенты.
- По завершении: коммит, затем PR в `main` через `gh pr create` с
  `Closes #N` в описании. Мержит PR и удаляет worktree оркестратор.

## Стек
TypeScript + HTML5 Canvas + Vite. Тесты — Vitest. Распознавание — алгоритм $P.
Python в MVP не используется (запланирован как сервер позже).

## Команды
- `npm run dev` — дев-сервер.
- `npm test` — юнит-тесты.
- `npm run build` — сборка.
