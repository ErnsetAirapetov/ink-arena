# InkArena — механика v2: 6 стихий, мульти-штрих, каст по пробелу (дизайн)

**Дата:** 2026-06-15
**Статус:** утверждён, готов к планированию реализации
**Предыдущая спека:** `2026-06-15-inkarena-mvp-design.md` (этот документ её развивает)

## 1. Что меняем и зачем

После теста MVP «рисовалка» понравилась. Дорабатываем ядро:

- расширяем набор глифов до **6 стихий**;
- разрешаем рисовать заклинание **несколькими линиями** (отпускание кнопки больше
  не кастует);
- каст переносим на **пробел**;
- **убираем учёт скорости рисования** (лишний контроль на этом этапе);
- силу считаем **просто по точности** совпадения с шаблоном; у комбо — средняя
  точность.

**Принцип баланса:** один каст = одно заклинание (нельзя заранее наштамповать
много глифов и слить разом).

## 2. Глифы 6 стихий

Формы намеренно максимально непохожи, чтобы `$P` уверенно их разделял. Глиф
может рисоваться одной или несколькими линиями — `$P` складывает все линии группы
в одно «облако точек».

| id | Стихия | Форма |
|----|--------|-------|
| `fire` | Огонь | △ треугольник вершиной вверх |
| `water` | Вода | ~ волна (синусоида) |
| `air` | Воздух | спираль/завиток |
| `earth` | Земля | □ квадрат |
| `lightning` | Молния | зигзаг (острые углы) |
| `shield` | Щит | ○ круг |

Глиф `arrow` из MVP удаляется (заменяется новым набором).

## 3. Ввод и каст

- Игрок рисует штрихи (мышь/палец). Каждый завершённый штрих **остаётся видимым
  на холсте** и накапливается в буфер до каста.
- **Пробел** запускает каст: распознавание накопленных линий, исполнение
  заклинания, затем **очистка буфера и холста**.
- Скорость рисования нигде не учитывается. (Естественный баланс: быстрее рисуешь
  — больше заклинаний в единицу времени; отдельный контроль не нужен.)

## 4. Сегментация линий в глифы (кластеризация по близости)

В момент каста накопленные штрихи группируются в кластеры по пространственной
близости: два штриха в одном кластере, если зазор между их ограничивающими
прямоугольниками меньше порога `clusterGapPx`. Используется одно-связная
(single-link) агломерация — транзитивно близкие штрихи попадают в одну группу.

Каждый кластер распознаётся как один глиф (все его точки — одно облако для `$P`).

## 5. Разрешение каста

На входе — список распознанных глифов (по одному на кластер), у каждого `score`
`0..1`. Результат каста (`CastOutcome`) — один из:

- **single** — одиночное заклинание: `{ kind: 'single', id, name, power }`.
- **combo** — комбо-заклинание: `{ kind: 'combo', id, name, power }`.
- **fizzle** — осечка: `{ kind: 'fizzle', reason }`.

Логика `resolveCast(results)`:

1. `results.length === 0` → fizzle («ничего не нарисовано»).
2. `results.length === 1`:
   - `score >= minScore` → single, `power = round(score * 100)`;
   - иначе → fizzle («слишком неточно»).
3. `results.length === 2`:
   - если любой `score < minScore` → fizzle («слишком неточно»);
   - иначе ищем комбо по двум id (порядок не важен): нашли → combo,
     `power = round(среднее(score1, score2) * 100)`; не нашли → fizzle
     («эти глифы не сочетаются»).
4. `results.length > 2` → fizzle («пока можно не больше двух глифов»).

## 6. Комбо (пространственные, порядок-независимые)

Комбо — два глифа, нарисованных вместе и скастованных разом. **Порядок не
важен**: `findCombo(a, b)` находит комбо при любом порядке пары. Таймера/окна
больше нет (механика стала пространственной, а не временно́й).

Стартовая таблица:

| id | Комбо | Из чего |
|----|-------|---------|
| `firestorm` | Огненный вихрь | огонь + воздух |
| `storm` | Шторм | вода + молния |
| `magma` | Магма | земля + огонь |
| `healing-barrier` | Лечащий барьер | вода + щит |

## 7. Сила

- Одиночное: `power = round(score * 100)`.
- Комбо: `power = round(среднее(score_глиф1, score_глиф2) * 100)`.

Других множителей нет (скорость убрана). Это сознательно простой старт.

## 8. Изменения в архитектуре

Новое:
- `recognition/clustering.ts` — `clusterStrokes(strokes, gapPx)` → группы штрихов.
  Чистая функция, юнит-тесты.
- `spells/cast.ts` — `resolveCast(results)` → `CastOutcome`. Чистая функция,
  юнит-тесты. Здесь же расчёт силы.

Меняется:
- `recognition/glyphs.ts` — новый набор из 6 глифов (без `arrow`).
- `spells/combo.ts` — `findCombo(a, b)` без таймера, порядок-независимый; новая
  таблица `COMBOS`. Класс `ComboTracker` удаляется.
- `spells/spell-system.ts` — упрощается/поглощается `cast.ts`: убирается скорость
  и `SpeedTier`. (Тип одиночного заклинания уезжает в `cast.ts`.)
- `config.ts` — убрать `speed` и `comboWindowMs`; добавить `clusterGapPx`.
- `drawing/stroke.ts` — без изменений в записи одного штриха; накопление многих
  штрихов держит `main.ts` (буфер завершённых `Stroke`).
- `drawing/canvas-renderer.ts` — без изменений (рисует переданные точки; `main`
  вызывает для каждого штриха буфера).
- `ui/hud.ts` — сообщения для single/combo/fizzle, без скорости.
- `main.ts` — копит штрихи, рисует все накопленные + текущий, ловит пробел,
  гоняет конвейер кластеризация → распознавание → `resolveCast` → эффект + HUD,
  очищает буфер после каста.

Геометрия: для кластеризации нужны границы штриха и зазор между штрихами —
добавить в `geometry.ts` хелперы `boundingBox(points)` и
`boxGap(a, b)` (либо разместить в `clustering.ts`, если используются только там).

## 9. Поток данных (v2)

1. pointerdown/move/up → `StrokeRecorder` → завершённый `Stroke` → в буфер `main`.
2. Каждый кадр: отрисовка всех штрихов буфера + текущего + частиц.
3. keydown «Space» → `clusterStrokes(buffer, clusterGapPx)` → для каждой группы
   `recognize(points, GLYPHS)` → массив `MatchResult` → `resolveCast(results)`.
4. По `CastOutcome`: `Hud` показывает результат, `EffectSystem.burst` рисует
   эффект (цвет по id), буфер и холст очищаются.

## 10. Тестирование

Юнит-тесты (Vitest) на чистые модули:
- `clustering`: близкие штрихи → одна группа; далёкие → разные; транзитивность.
- `recognizer`: каждый из 6 эталонов узнаётся как сам себя; формы различимы
  (top-матч верный).
- `combo`: `findCombo` находит пару в любом порядке; несуществующая пара → null.
- `cast`: все ветки `resolveCast` (0/1/2/>2 кластеров, ниже порога, комбо/не
  комбо, расчёт силы и средней силы).

Визуальное (рисование, накопление линий, очистка по пробелу, эффекты) —
проверяется вручную в браузере.

## 11. Вне области (YAGNI, в roadmap)

- Сложные заклинания из 5–6+ глифов, где тип определяется размером, направлением
  и взаимным расположением глифов (как в аниме «Академия колдовских колпаков»).
- Кулдауны и мана.
- Красивая стилизация глифов под WHA (сейчас — простые формы).
- Соперник, HP, онлайн (как и раньше).
