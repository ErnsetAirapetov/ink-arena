# Боевая система — Фаза C2: глифы баф/дебаф и стихийные эффекты (дизайн)

**Дата:** 2026-06-15
**Статус:** утверждён, готов к планированию
**Связано:** `2026-06-15-combat-phase-c-design.md` (под-фаза C2), опирается на C1 (статусы)

## 1. Цель

Завершить систему статусов: добавить глифы-модификаторы баф (`/`) и дебаф (`\`),
три стихийных эффекта (горение, уязвимость, усиление) с длительностью, индикаторы
активных статусов. C1 уже дал общий список статусов и щит-пул.

## 2. Новые глифы

Два новых модификатор-глифа, одной линией, различимы по наклону ($P не
инвариантен к повороту):
- **`buff` = `/`** (восходящая черта): точки `[(0, 100), (100, 0)]`;
- **`debuff` = `\`** (нисходящая): точки `[(0, 0), (100, 100)]`.

Добавляются в `GLYPHS` — автоматически становятся гост-карточками в подложке.
Имена: «Усиление» (buff), «Проклятие» (debuff) — как подписи карточек.

## 3. Грамматика (расширение `parseSpell`)

Модификаторы теперь три: `shield`, `buff`, `debuff`. Эффект определяется парой
модификатор+стихия по таблице связок.

| Нарисовано | Spell |
|------------|-------|
| `buff` + стихия (связка есть) | `buff(element, effect)` — аура на себя |
| `debuff` + стихия (связка есть) | `attack(element, effect)` — атака + дебаф |
| `buff`/`debuff` в одиночку | осечка «Модификатору нужна стихия» |
| модификатор + стихия без связки | осечка «Эффект ещё не изучен» |
| `shield` / комбо / атака — без изменений | как в фазе B/C1 |

Определены 3 связки (`EFFECT_TABLE`):
- `debuff + fire → burn` (Горение);
- `debuff + earth → vulnerable` (Уязвимость);
- `buff + lightning → empower` (Усиление).

Типы заклинаний:

```ts
type EffectKind = 'burn' | 'vulnerable' | 'empower';
type Spell =
  | { kind: 'attack'; element: string; name: string; power: number; effect?: EffectKind }
  | { kind: 'combo'; id: string; name: string; power: number }
  | { kind: 'shield'; element: string | null; name: string; power: number }
  | { kind: 'buff'; element: string; effect: EffectKind; name: string; power: number }
  | { kind: 'fizzle'; reason: string };
```

## 4. Эффекты (числа в `config`)

- **Горение** (`burn`, дебаф 🔥): `dps` урона/сек на `durationMs`. Старт: `6`/сек × `4000`.
- **Уязвимость** (`vulnerable`, дебаф ⛰): цель получает `+extra` доля входящего на
  `durationMs`. Старт: `0.4` × `5000`.
- **Усиление** (`empower`, баф ⚡): исходящие атаки кастера `×(1+bonus)` на
  `durationMs`. Старт: `0.4` × `5000`.

Повторное наложение того же эффекта **обновляет длительность** (по величине не
копится — вне области, раздел 8).

## 5. Применение в бою

- `buff` → `addEffect` к игроку (усиление применяется к себе сразу).
- `attack` с `effect` → снаряд несёт `effect`; по прилёте `addEffect` к цели
  (манекену) — горение/уязвимость.
- Каждый кадр `tickStatuses(c.statuses, dt)` возвращает `{ statuses, dotDamage }`;
  `dotDamage` (горение) списывается с HP бойца.
- **Уязвимость** усиливает входящий урон в `absorbIncoming` (до щита):
  `raw' = raw × (1 + Σ extra)`.
- **Усиление** множит исходящий урон игрока: в `cast()` урон атаки
  `× outgoingMultiplier(player.statuses)`.

## 6. Модули

Меняется:
- `recognition/glyphs.ts` — глифы `buff` (`/`), `debuff` (`\`).
- `spells/spell-types.ts` — `EffectKind` импортируется из `status`; таблица
  `EFFECT_TABLE` (модификатор+стихия → эффект); типы `buff`/атака-с-эффектом;
  разбор модификаторов.
- `combat/status.ts`:
  - `addEffect(statuses, status)` — добавить баф/дебаф (заменить существующий того
    же `kind`, обновив длительность);
  - `tickStatuses` меняет сигнатуру на `→ { statuses: Status[]; dotDamage: number }`
    (накопить урон от `burn`);
  - `absorbIncoming` учитывает `vulnerable` (усиливает `raw` до щита);
  - `outgoingMultiplier(statuses) → number` — произведение `(1 + bonus)` от `empower`;
  - `makeEffect(kind: EffectKind) → Status` — собрать статус из чисел `config`.
- `combat/projectile.ts` — `SpawnOpts`/`Arrival`/`Projectile` получают опц.
  `effect?: EffectKind`.
- `combat/scene.ts` — индикаторы статусов: чипы под HP-баром бойца (щит/горение/
  уязвимость/усиление + остаток секунд).
- `ui/hud.ts` — сообщения для бафа и атаки-с-дебафом.
- `ui/guide.ts` — легенда эффектов (связка → эффект).
- `main.ts` — маршрутизация бафа/дебафа, DoT из `tickStatuses`, множитель
  исходящего урона, наложение эффекта по прилёте снаряда.
- `config.ts` — поля эффектов (`burnDps`, `burnMs`, `vulnerableExtra`,
  `vulnerableMs`, `empowerBonus`, `empowerMs`).

## 7. Тестирование

- `status`: `addEffect` (обновление длительности того же эффекта); `tickStatuses`
  возвращает `dotDamage` от `burn` и снимает истёкшее; `outgoingMultiplier`
  (произведение усилений); `absorbIncoming` с `vulnerable` (усиление до щита).
- `spell-types`: баф+стихия (связка), дебаф+стихия (атака с эффектом), одиночный
  модификатор → осечка, неизученная связка → осечка.
- `recognizer`: `/` и `\` узнаются как сами себя.
- Визуальное (чипы статусов, DoT, легенда) — вручную в браузере.

## 8. Вне области (roadmap)

- Остальные связки модификатор+стихия (все 5 стихий × баф/дебаф).
- Стак эффектов по величине (сейчас только обновление длительности).
- Дебафы от атак манекена (манекен бьёт без эффектов).
- Свет/тьма; касты из 3+ глифов.
