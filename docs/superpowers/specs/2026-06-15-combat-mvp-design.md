# Дизайн: минимальный бой (Combat MVP)

Дата: 2026-06-15
Ветка: `feat/fight`

## Цель
Добавить минимальную боевую механику поверх существующей песочницы:
персонаж игрока, тренировочный манекен с HP, нанесение урона заклинанием и
простая анимация попадания. Манекен — для бесконечной тренировки.

## Решения
- **Смерть манекена:** авто-респавн — при HP=0 анимация смерти, через
  `respawnMs` восстановление полного HP.
- **Игрок:** статичная фигура-кастер слева. Без HP и урона по игроку (задел
  на будущего противника, но не в этом MVP).
- **Урон:** зависит только от точности заклинания (`spell.power`). Скорость и
  комбо на урон пока не влияют.
- **Анимация попадания:** вспышка (мигание красным) + тряска + всплывающая
  цифра урона. Существующий взрыв частиц переносится на позицию манекена.

## Архитектура

### `src/combat/combat.ts` — чистый модуль (TDD)
Логика HP и урона, без DOM и времени.

```ts
interface Combatant { hp: number; maxHp: number; alive: boolean }

createCombatant(maxHp: number): Combatant      // полный HP, alive=true
damageFor(spell: Spell): number                // round(power * CONFIG.combat.damagePerPower)
applyDamage(c: Combatant, amount: number): Combatant  // hp обрезан по [0, maxHp], alive = hp>0
respawn(c: Combatant): Combatant               // hp=maxHp, alive=true
```

Функции чистые: возвращают новый `Combatant`, не мутируют вход.

### `src/combat/scene.ts` — визуальный модуль (проверка вручную)
Рисует сцену и держит транзитное состояние анимаций. Логическое состояние HP
сюда передаётся снаружи, не хранится.

Состояние анимаций:
- `flash` (0..1) — интенсивность мигания, затухает по времени;
- `shake` — амплитуда смещения манекена, затухает;
- `floaters` — всплывающие цифры урона `{ value, x, y, life }`;
- состояние смерти/появления (плавное затухание манекена при HP=0).

API:
```ts
class CombatScene {
  hit(amount: number): void                    // запустить вспышку, тряску, всплывающую цифру
  update(dtMs: number): void                   // двигать таймеры анимаций
  draw(ctx, dummy: Combatant, size: {w,h}): void  // игрок слева, манекен справа, HP-бар
}
```

## Поток данных
```
ввод указателя → StrokeRecorder → recognize() → buildSpell()
  → ComboTracker.push()
  → [успешное заклинание] damageFor(spell) → applyDamage(dummy)
  → scene.hit(dmg) + effects.burst(на позиции манекена)
  → [HP=0] запуск таймера респавна → respawn(dummy)
```

## Баланс — `src/config.ts`, новый блок `combat`
```ts
combat: {
  dummyHp: 100,        // HP манекена
  damagePerPower: 0.6, // множитель урона от точности (power 100 → 60)
  respawnMs: 1500,     // задержка авто-респавна после смерти
}
```
При power ~70–100 урон ~42–60 → примерно 2 точных попадания до смерти.

## Изменения в существующих файлах
- `src/config.ts` — добавить блок `combat` (общий файл, минимальная вставка).
- `src/main.ts` — создать манекен `createCombatant(CONFIG.combat.dummyHp)`,
  создать `CombatScene`, на успешном заклинании наносить урон и анимировать,
  отрисовывать сцену в игровом цикле, переносить `effects.burst` на манекен,
  обрабатывать таймер респавна. Общий файл с агентом по рисованию — вставки
  делать отдельными блоками для упрощения merge.
- `docs/architecture.md` — описать модуль `combat`.
