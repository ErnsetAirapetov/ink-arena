# Боевая система — Фаза B: двусторонний бой и грамматика защиты (дизайн)

**Дата:** 2026-06-15
**Статус:** утверждён, готов к планированию
**Связано:** `2026-06-15-combat-system-design.md` (фаза B), опирается на фазу A (сродство)

## 1. Цель и мотивация

Защита, щиты и бафы сейчас бессмысленны: манекен не атакует игрока, входящего
урона нет. Фаза B делает бой **двусторонним** — манекен бьёт в ответ, у игрока
есть HP, а щиты реально снижают урон. Это первое реальное применение сродства
(фаза A) на защите и фундамент для статусов (фаза C).

Заодно вводится **типизованный парсер заклинаний** (грамматика): щит становится
модификатором защиты, появляются базовый и стихийный щиты.

## 2. Двусторонний бой

- **HP игрока.** У игрока появляется `hp`/`maxHp`/`alive` (сейчас только таймер
  щита). `maxHp` — в `config`. Смерть → авто-респавн полным HP после задержки
  (песочница, без гейм-овера).
- **ИИ манекена** (цикл-машина состояний):
  1. *телеграф*: над манекеном загорается иконка стихии следующей атаки на
     `telegraphMs` (~1500);
  2. *выстрел*: снаряд летит по игроку с этой стихией и уроном `dummyDamage`;
  3. *пауза* `dummyAttackIntervalMs` (~3000), затем снова.
  - Стихия атаки выбирается из 5 (последовательно по кругу — детерминированно,
    тестируемо; без `Math.random`).
- Снаряды летают в обе стороны: переиспользуется `ProjectileSystem`. Снаряд
  манекена несёт `targetSide: 'player'`, чтобы по прилёте бить игрока.

## 3. Щиты, снижающие урон

Щит в фазе B остаётся **таймерным** (прочность и стак — фаза C).

- Активный щит снижает входящий урон на долю `shieldBlock` (~0.6).
- **Стихийный щит** учитывает сродство:
  `blockFraction = clamp(shieldBlock × affinity(shieldElement, attackElement), 0, 0.95)`.
  - щит силён против стихии атаки (`affinity 1.5`) → блок ~0.9;
  - слаб (`affinity 0.66`) → ~0.4;
  - базовый щит (без стихии) → `affinity = 1.0` → ровно `shieldBlock`.
- Финальный урон по игроку: `round(raw × (1 − blockFraction))`.

## 4. Грамматика заклинаний (типизованный парсер)

Новый модуль `spells/spell-types.ts` — `parseSpell(results) → Spell`, заменяет
`resolveCast`. Типы фазы B (без бафа/дебафа — фаза C):

```ts
type Spell =
  | { kind: 'attack'; element: string; name: string; power: number }
  | { kind: 'combo'; id: string; name: string; power: number }
  | { kind: 'shield'; element: string | null; name: string; power: number }
  | { kind: 'fizzle'; reason: string };
```

Разбор распознанных глифов (по их `id`): глиф `shield` (круг) трактуется как
**модификатор защиты**, остальные пять — как стихии.

| Нарисовано | Spell |
|------------|-------|
| 0 глифов / >2 / ниже порога | `fizzle` |
| 1 стихия | `attack(element)`, `power = round(score×100)` |
| 1 `shield` | `shield(element: null)` — базовый |
| `shield` + стихия | `shield(element)` — стихийный |
| стихия + стихия (есть комбо) | `combo` (сила со сродством, как в фазе A) |
| стихия + стихия (нет комбо) / два `shield` / прочее | `fizzle` |

Параметры порога/числа глифов — те же, что в нынешнем `resolveCast`.

**Последствие:** комбо `healing-barrier` (вода+щит) **удаляется** из `COMBOS` —
теперь «вода+щит» парсится как стихийный водяной щит. Комбо `firestorm`, `storm`,
`magma` не меняются.

## 5. Маршрутизация в бою (`main.ts`)

- `attack` / `combo` → снаряд в манекен (как сейчас; размер/урон/полёт прежние).
- `shield(element|null)` → поднять щит игроку с этой стихией на `shieldMs`.
- Каждый кадр: ИИ манекена тикает; при выстреле манекена — снаряд в игрока; по
  прилёте — `applyDamageToPlayer` с учётом блока; смерть игрока → респавн по
  таймеру.

## 6. Модель данных и модули

Меняется:
- `combat/player.ts` — `Player { hp, maxHp, alive, shieldMs, shieldElement }`;
  `createPlayer(maxHp)`, `castShield(p, durationMs, element)`,
  `tickPlayer(p, dtMs)` (сбрасывает `shieldElement` при истечении),
  `applyDamageToPlayer(p, rawDamage, attackElement)`, `respawnPlayer(p)`,
  `isShielded(p)`.
- `combat/combat.ts` — добавить `blockedDamage(shieldElement, raw, attackElement)`
  (чистая формула блока из раздела 3).
- `spells/combo.ts` — убрать `healing-barrier` из `COMBOS`.
- `combat/scene.ts` — HP-бар игрока; иконка-телеграф над манекеном.
- `ui/hud.ts` — сообщение для щита с указанием стихии; уже есть `showShield`,
  адаптировать под стихию.
- `main.ts` — использование `parseSpell`, маршрутизация защиты/атаки, цикл ИИ.

Создаётся:
- `spells/spell-types.ts` — `Spell`, `parseSpell`.
- `combat/dummy-ai.ts` — `DummyAi` (машина состояний телеграф→выстрел→пауза):
  `createDummyAi()`, `tickDummyAi(state, dtMs) → { state, fire?: { element } }`,
  `telegraphElement(state) | null`.

Адаптируется/удаляется: `spells/cast.ts` — логика переезжает в `spell-types.ts`;
сам `resolveCast` удаляется (его тесты переносятся/обновляются под `parseSpell`).

## 7. Конфиг (новые поля в `config.combat`)

- `playerHp` (≈100), `playerRespawnMs` (≈1500);
- `dummyDamage` (≈15), `telegraphMs` (≈1500), `dummyAttackIntervalMs` (≈3000);
- `shieldBlock` (≈0.6), `maxBlockFraction` (0.95).

## 8. Тестирование

- `spell-types`: все ветки `parseSpell` — атака, комбо (со сродством),
  базовый щит, стихийный щит, осечки (два щита, нет комбо, >2, ниже порога).
- `blockedDamage`: базовый блок; стихийный сильный/слабый; обрезка `[0, 0.95]`;
  без щита (element-less не вызывается — функция чистая, тест на формулу).
- `combat.applyDamageToPlayer`: урон сквозь щит уменьшен; смерть при 0 HP;
  отсутствие щита → полный урон.
- `dummy-ai`: переходы по таймерам (телеграф→выстрел→пауза), последовательный
  выбор стихии, событие `fire` ровно в момент выстрела.
- Визуальное (снаряды в игрока, иконка-телеграф, HP-бар игрока) — вручную.

## 9. Вне области (следующие фазы)

- Бафы/дебафы и их глифы; стихийные эффекты (фаза C).
- Прочность и стак щита (фаза C).
- Полноценный PvP-оппонент с осмысленным выбором контр-стихий и собственными
  щитами (фаза D и далее).
