# InkArena — боевая система: стихии, грамматика, статусы (дизайн)

**Дата:** 2026-06-15
**Статус:** утверждён, готов к планированию первой фазы
**Связано:** развивает текущий бой (`combat/*`) и каст (`spells/*`)

## 1. Цель

Превратить «скелет» боя в полноценную систему магии:
- стихии связаны как «камень-ножницы-бумага» (сила/слабость → модификаторы);
- заклинания делятся на типы (атака / щит / баф / дебаф), тип задают
  **модификатор-глифы**;
- атаки летят снарядом и могут накладывать дебаф; защита/баф применяются к себе;
- бафы/дебафы имеют длительность; щит — прочность и стакается.

Документ описывает **всю систему** концептуально; реализация — по фазам (раздел 8),
каждая фаза получит свой план.

## 2. Что уже есть (отправная точка)

- `combat/combat.ts`: `Combatant {hp,maxHp,alive}`, урон/респавн; `sizeFactor`,
  `damageFor(sizeFactor, accuracy)`, `flightTimeMs`.
- `combat/player.ts`: `Player {shieldMs}` — щит как таймер (без прочности/стака).
- `combat/projectile.ts`: снаряды дугой, урон по прилёте.
- `spells/cast.ts`: `resolveCast(results) → {single|combo|fizzle, id, name, power}`.
- Глифы: 5 стихий + круг «щит» (сейчас считается элементом).

## 3. Элементы и сродство (пентаграмма)

Пять стихий, чистый 5-цикл: каждая бьёт 2 и слаба против 2 (нейтральных нет).

| Стихия | Бьёт (×1.5) | Слаба против (×0.66) |
|--------|-------------|----------------------|
| `fire` Огонь | `air`, `earth` | `lightning`, `water` |
| `air` Воздух | `earth`, `lightning` | `fire`, `water` |
| `earth` Земля | `lightning`, `water` | `fire`, `air` |
| `lightning` Молния | `water`, `fire` | `air`, `earth` |
| `water` Вода | `fire`, `air` | `lightning`, `earth` |

- `affinity(attacker, defender)` → `1.5` (сильно), `0.66` (слабо). При совпадении
  стихий — `1.0`. Множители (`strongMult`, `weakMult`) — в `config.ts`.
- Флейвор связей можно менять, структура цикла — нет (иначе ломается баланс).

## 4. Грамматика заклинаний

Глифы делятся на:
- **стихии (5):** `fire`, `water`, `air`, `earth`, `lightning`;
- **модификаторы (3):** `shield` (защита, существующий круг), `buff`, `debuff`
  (два новых различимых глифа; конкретные формы определяются в фазе B по тем же
  принципам, что и прежние глифы).

Каст — 1–2 глифа (3+ откладывается). Разбор распознанных глифов в типизованное
заклинание:

| Нарисовано | Результат |
|------------|-----------|
| 1 стихия | `Attack(element)` — атака (дефолт) |
| стихия + стихия | `ComboAttack` — как сейчас, плюс бонус сродства |
| `shield` | `Shield(basic)` — базовый щит без стихии |
| `shield` + стихия | `Shield(element)` — стихийный щит |
| `buff` + стихия | `Buff(element)` — аура на себя |
| `debuff` + стихия | `Attack(element)` + накладывает `Debuff(element)` |
| два модификатора / прочее | осечка |

«По умолчанию одиночная стихия — атака; щит — всегда защита» (требование) —
выполняется.

## 5. Где работает сродство

1. **Атака против щита:** `affinity(attackElement, shieldElement)` множит урон,
   который атака снимает с прочности щита (сильная атака пробивает быстрее).
2. **Комбо:** если две стихии комбо в «сильной» связи — бонус к силе (множитель).
3. **Защита:** стихийный щит держит лучше против стихий, против которых он силён
   (та же `affinity`, с точки зрения щита).
4. **Голая цель** (без активного щита/ауры): сродство не применяется — обычный
   урон. Глубина раскрывается в PvP, когда оба используют стихийную защиту.

## 6. Статусы (бафы / дебафы / щит)

Общая модель: у бойца список активных статусов, у каждого — длительность (или
прочность для щита), тик каждый кадр, снятие по истечении.

### 6.1 Щит
- Поле: `{ kind:'shield', element?, absorb, durationMs }`.
- Поглощает входящий урон до `absorb`; при 0 прочности или 0 времени — спадает.
- **Стак:** повторный каст щита добавляет `absorb` и продлевает `durationMs`
  (с верхними кэпами `maxShieldAbsorb`, `maxShieldMs` в `config.ts`).
- Стихийный щит: поглощаемый урон корректируется `affinity` стихии атаки против
  стихии щита.

### 6.2 Бафы и дебафы
- Поле: `{ kind:'buff'|'debuff', element, effect, magnitude, durationMs }`.
- Эффекты **привязаны к стихии**. В первой фазе реализуем **2-3**:
  - дебаф `burn` (🔥): DoT — `magnitude` урона в секунду в течение `durationMs`;
  - дебаф `vulnerable` (⛰): цель получает `+magnitude%` входящего урона;
  - баф `empower` (🔥): исходящие атаки кастера `+magnitude%` урона.
- Остальные стихии получают свои эффекты в следующих фазах (раздел 8, фаза C+).

## 7. Модель данных (модули)

- `combat/elements.ts` — `type Element`, таблица сродства, `affinity(att, def)`.
  Чистая, юнит-тесты.
- `spells/spell-types.ts` — `parseSpell(results) → Spell`, где
  `Spell = Attack | ComboAttack | Shield | Buff` (атака с дебафом — поле на
  `Attack`). Чистая, юнит-тесты. Заменяет/оборачивает текущий `resolveCast`.
- `combat/status.ts` — `Status`, `tickStatuses`, `applyShieldAbsorb`,
  `addShield` (стак), `applyBuffDebuff`. Чистая, юнит-тесты.
- Адаптируются: `combat/combat.ts`, `combat/player.ts` (статусы вместо
  `shieldMs`), `spells/cast.ts`, `main.ts`, `ui/hud.ts`, `recognition/glyphs.ts`
  (новые модификатор-глифы), `config.ts`.

## 8. Фазы реализации

Каждая фаза — отдельный спек/план/реализация, оставляет игру рабочей.

- **Фаза A — Сродство.** `combat/elements.ts` + множитель; подключить к урону
  атаки против щита и к бонусу комбо. (Минимум зависимостей, чистая основа.)
- **Фаза B — Грамматика.** `spells/spell-types.ts`: разбор глифов в типизованное
  заклинание; щит → модификатор; добавить глифы `buff`/`debuff`; обновить парсер
  и подложку-обводку.
- **Фаза C — Статусы.** `combat/status.ts`: общая система; щит → прочность+стак;
  2-3 стихийных эффекта (burn, vulnerable, empower).
- **Фаза D — Сборка.** Связать грамматику + сродство + статусы в бою: атака
  накладывает дебаф, баф применяется к себе, щит поглощает с учётом сродства;
  показать активные статусы в HUD.

## 9. Тестирование

- `elements`: `affinity` для всех пар (симметрия сильно/слабо, совпадение → 1.0).
- `spell-types`: все ветки разбора (атака/комбо/щит/стихийный щит/баф/дебаф/осечка).
- `status`: тик длительности и снятие; поглощение щитом; стак (рост absorb и
  времени, кэпы); DoT-урон по тикам; модификаторы урона (vulnerable, empower).
- Визуальное (снаряды, индикаторы статусов, подложка) — вручную в браузере.

## 10. Вне области (roadmap)

- Свет/тьма как отдельная пара.
- Касты из 3+ глифов и сложные модификаторы (размер/направление/расположение).
- Полный каталог стихийных эффектов (все 5 стихий × баф/дебаф).
- Полноценный PvP-оппонент (сейчас цель — манекен).
