# Архитектура

Целевая архитектура нового движка (Р38). Направление закреплено; детали
фиксируются здесь по мере принятия решений. Текущий код в `src/` —
прототип-песочница, написанный до фазы 0; его архитектура — в архиве
`prototype/architecture.md`; он будет переписан.

## Принципы

1. **Детерминированная симуляция ядром.** Вся игровая логика — чистая
   детерминированная функция (состояние, ввод) → состояние. Это даёт:
   реплеи как запись входных событий (Р12), честную проверку на будущем
   авторитарном сервере, дешёвые режимы поверх той же симуляции (пошаговый
   «бой по переписке», Р22) и простое тестирование ядра.
2. **Ядро не знает о платформе.** Симуляция и распознавание не зависят от
   Canvas/DOM; клиент — тонкий слой рендера и ввода.
3. **Web-first, mobile-first.** Платформа доставки — веб (кратчайший путь
   к MVP, Р14); первый пользовательский сценарий — телефон в портрете
   (Р13, Р35). Мобильная упаковка: PWA, затем Capacitor.
4. **Баланс — данные, не код.** Все числа — в конфиге (сейчас
   `src/config.ts`); код только читает их.

## Структура (целевая)

Monorepo на TypeScript:

```
core/    ← детерминированная симуляция: правила боя, чернила, статусы,
           раунды; распознавание ($P) и парсер композиции «ядро и
           орбиты» (Р21, Р26)
client/  ← рендер (Canvas), ввод (палец / мышь / стилус), UI, звук
server/  ← ПОЗЖЕ (к альфе): авторитарный сервер поверх того же core.
           Реалтайм не twitch-уровня: снаряды летят секунды,
           100–200 мс задержки достаточно (Р22)
```

Поток данных: ввод → штрихи → распознавание глифов → парсинг композиции →
валидное заклинание → событие симуляции → новое состояние → рендер.

## Технологический стек

- Язык: TypeScript везде (ядро, клиент, позже сервер — чтобы шарить core).
- Сборка: Vite. Рендер: HTML5 Canvas. Тесты: Vitest.
- TDD для чистых модулей (всё ядро); визуальные модули проверяются
  вручную в браузере.
- Распознавание: алгоритм $P (point-cloud) + кластеризация штрихов для
  разбора композиции. Детали → `tech/recognition.md` (появится).
- Python в MVP не используется (сервер тоже будет TypeScript).

## Внешние сервисы и зависимости

Сейчас — никаких. Появятся не раньше альфы (сеть, аккаунты, лидерборды);
