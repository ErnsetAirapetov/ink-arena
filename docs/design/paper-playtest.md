# Бумажная проверка грамматики v0 — материалы плейтеста

Расходник эксперимента 2 из Р36: до переписывания движка проверяем на бумаге,
однозначно ли парсятся правила компоновки Р26 и ощущается ли cost-модель
«больше орбит — дороже, дольше, сильнее» (Р21) справедливой на корпусе Р32.
Это кухня фазы 0 (`docs/design/`), не источник правды об игре — источник
правды остаётся в `docs/game/spellcraft.md` и `docs/game/combat.md`.

Все числа здесь — только качественные соотношения (слабее/сильнее,
дёшево/средне/дорого). Ни одной цифры из `src/config.ts` в этом файле нет и
не будет: конкретные значения баланса эксперимент не проверяет, он проверяет
понятность правил и ощущение справедливости.

Печатать по разделам «Лист 1–3» отдельными страницами, раздать каждому
игроку. «Протокол партии» — памятка ведущему (им может быть один из
игроков).

## Лист 1 — Пентаграмма стихий (Р24)

| Стихия | Бьёт | Слаба против |
|--------|------|--------------|
| 🔥 Огонь | 🌪 воздух, ⛰ земля | 💧 вода, ⚡ молния |
| 💧 Вода | 🔥 огонь, 🌪 воздух | ⛰ земля, ⚡ молния |
| 🌪 Воздух | ⛰ земля, ⚡ молния | 🔥 огонь, 💧 вода |
| ⛰ Земля | ⚡ молния, 💧 вода | 🔥 огонь, 🌪 воздух |
| ⚡ Молния | 💧 вода, 🔥 огонь | 🌪 воздух, ⛰ земля |

Мнемоника — один цикл по кругу, каждая стихия бьёт следующие ДВЕ:

```
Огонь → Воздух → Земля → Молния → Вода → (снова Огонь)
```

Ничьих по стихиям нет: для любой пары стихий ровно одна бьёт другую. Состав
фиксирован (Р24) — при разборе каста сверяйтесь только с этой таблицей, не
угадывайте по интуиции.

## Лист 2 — Шпаргалка правил компоновки (Р26)

### Словарь глифов (Р25)

| Глиф | Роль |
|------|------|
| 🔥 💧 🌪 ⛰ ⚡ | стихия — ядро заклинания, ровно одна |
| без глагола | атака (дефолт) — стихия в одиночку |
| дуга `⌒` | защита (щит) |
| спираль `🌀` | хил |
| `/` | баф (усиление) |
| `\` | дебаф (ослабление) |
| круг вокруг всего | не глиф — жест «каст ушёл» (Р11, Р27), бесплатный |

### Шесть правил

1. **Граница.** Заклинание — всё, что внутри замыкающего круга.
2. **Ядро.** В центре ровно один глиф стихии, и он самый крупный.
3. **Орбиты.** Вокруг ядра — от нуля до N глаголов-модификаторов.
4. **Дефолт.** Нет глагола-модификатора — базовая атака этой стихией.
5. **Вес.** Относительный размер модификатора = его вес в заклинании.
   Крупнее нарисован — сильнее действует.
6. **Осечка.** Нарушение структуры — рисунок не стал заклинанием.

### Три вида осечки — примеры

**А) Две стихии — нет единственного ядра**

```
   ╭───────────────╮
  │   🔥      💧      │   осечка: два ядра одного размера,
  │                  │   непонятно, какое из них главное
   ╰───────────────╯
```

**Б) Глагол без стихии — нечего модифицировать**

```
   ╭───────────╮
  │     ⌒        │   осечка: дуга есть, стихии в центре нет
   ╰───────────╯
```

**В) Глиф на границе круга — нельзя решить, внутри он или снаружи**

```
   ╭───────────╮
  │    🔥        │
  │           ⌒ │ ← дуга рассечена линией круга пополам:
   ╰──────────┼──╯   осечка, глиф не однозначно внутри
```

Всё, что прошло правила 1–5 и не попало ни в один из трёх случаев выше —
валидное заклинание, даже нарисованное впервые (Р9). Спорный случай — не
повод спорить «на глаз»: если после сверки с этими тремя примерами игроки
всё ещё не согласны, случай уходит в чек-лист наблюдений как есть, без
натягивания решения.

Контекст, не проверяется на бумаге: в игре осечка сжигает чернила,
потраченные на рисунок (Р29) — на бумаге чернила не считаем (см. протокол),
достаточно того, что рисунок не состоялся как заклинание.

## Лист 3 — Эталонный корпус (Р32)

Пять заклинаний ниже — ориентир, не обязаловка: корпус показывает диапазон
композиций, но правила Р26 разрешают любую, что им подчиняется (Р9). Роль и
стоимость — качественные, из Р32.

**1. Искра** — быстрый дешёвый снаряд, «джеб». Стоимость: дёшево.

```
   ╭───────╮
  │   ⚡     │   маленькое ядро, без глаголов
   ╰───────╯
```
Разбор: ядро — молния (правило 2); орбит нет (правило 3); без глагола —
дефолт-атака (правило 4); размер минимальный.

**2. Огненная стена** — щит со сродством огня. Стоимость: средне (щит
дешевле атаки той же силы).

```
   ╭─────────────╮
  │      ⌒         │   орбита: дуга-защита
  │    🔥           │   ядро крупнее дуги
   ╰──────────────╯
```
Разбор: ядро — огонь, самое крупное (правило 2); одна орбита — глагол
защиты (правило 3); итог — щит, а не атака, потому что глагол есть
(правило 4 не срабатывает).

**3. Родник** — хил себе, сила от точности. Стоимость: средне.

```
   ╭─────────────╮
  │      🌀         │   орбита: спираль-хил
  │    💧           │   ядро крупнее спирали
   ╰──────────────╯
```
Разбор: ядро — вода; орбита — глагол хила; вес модификатора умеренный
(правило 5) — спираль не доминирует над ядром.

**4. Проклятие камня** — снаряд, вешающий уязвимость (сетап перед своим
нюком). Стоимость: средне.

```
   ╭─────────────╮
  │      \          │   орбита: дебаф
  │    ⛰           │   ядро крупнее дебафа
   ╰──────────────╯
```
Разбор: ядро — земля; орбита — глагол дебафа; атака с довеском, а не чистый
дефолт, потому что глагол присутствует.

**5. Гнев бури** — дорогой медленный нюк, ставка раунда: долго рисуется и
долго летит, телеграфирует себя. Стоимость: дорого.

```
   ╭───────────────────╮
  │    /            /      │   две орбиты: два бафа
  │         ⚡              │   крупное ядро — больше любого модификатора,
   ╰────────────────────╯   но композиция в целом заметно крупнее «Искры»
```
Разбор: ядро — молния, крупное (правило 2, оно всё равно больше каждого
`/` по отдельности); две орбиты (правило 3) — двойной вес по правилу 5;
именно две орбиты и больший общий размер задают cost-модель «дороже и
дольше рисовать» (Р21) — это главное, что проверяет эксперимент на этом
эталоне.

## Протокол партии (памятка ведущему)

### Материалы

- Листы 1–3 этого документа (по экземпляру на игрока).
- Чистая бумага для рисования — свежий лист каждому на каждый раунд (Р28).
- Ручка/маркер на игрока.
- Что-нибудь, чем экранировать рисунок от соперника, пока он не закрыт
  кругом (папка, книга, ладонь) — бумажный аналог скрытого холста (Р16).
- Таймер (телефон) — 60–90 с на раунд, 5–10 с передышка (Р22).
- Лист-полоска HP на игрока — рисуется заново каждый раунд (см. ниже).

### Ход раунда

1. Ведущий объявляет начало раунда, оба рисуют новую полоску HP (одинаковой
   длины у обоих — «полный HP», Р28) и запускает таймер.
2. Оба игрока одновременно рисуют заклинания на своих листах, экранируя
   рисунок от соперника, пока не закрыли круг (Р16). Ходов и очереди нет —
   каждый рисует в своём темпе (Р22).
3. Как только игрок замыкает круг вокруг композиции (Р27) — это каст: он
   говорит «Есть!» и открывает лист. Не замкнул круг или зацепил лишнее —
   каст не ушёл, это не осечка, просто рисунок продолжается (Р27).
4. Сверяют рисунок с Листом 2 (правила Р26): валиден или осечка одного из
   трёх видов. Если оба игрока молча согласны — записывать не нужно; если
   был спор или пауза «а это как считать?» — это идёт в чек-лист.
5. Если валидно — по Листу 3 (или по правилам, если рисунок новый и в
   корпус не входит) определяют стихию и глаголы.
6. Смотрят Лист 1 (пентаграмма): бьёт ли стихия атаки стихию щита/цели —
   контрпик усиливает эффект, невыгодная стихия ослабляет его.
7. Определяют качественную силу удара/эффекта: смотрите правило веса
   (крупнее модификатор — сильнее, Р26.5) и стоимость эталона-ориентира
   (дёшево/средне/дорого, Р32) как стартовый уровень интенсивности,
   изменённый пентаграммой на шаге 6.
8. Если у цели есть свежий щит (дуга той же или контрящей стихии) — щит
   поглощает удар полностью («держит») или частично («пробит») — решают на
   глаз по тому же принципу веса и пентаграмме.
9. Отмечают результат на полоске HP цели (см. «Как считать урон» ниже).
10. Игроки сразу начинают рисовать следующее заклинание — раунд идёт
    непрерывно, до сигнала таймера или до нокаута.
11. Раунд заканчивается по Р41:
    - **нокаут** — у кого полоска HP закончилась, тот раунд проиграл;
    - **время вышло, оба живы** — раунд берёт тот, у кого полоска HP
      длиннее по остатку «на глаз»;
    - **остаток равный** — внезапная смерть: быстрый обмен ещё по одному
      заклинанию каждый, первый чистый удар решает;
    - **оба дошли до нуля в один и тот же обмен** — раунд ничейный, очко
      никому, играется следующий (Р41).
12. Между раундами — пауза 5–10 с (Р22). Новый раунд — с чистого листа:
    HP и чернила (на бумаге чернила не считаем) не переносятся; переносится
    только счёт матча и то, что игроки запомнили друг о друге (Р28).
13. Матч — до двух побед из трёх раундов (bo3, Р6/Р22).

### Как считать урон/щиты — качественно (без чисел)

Полоска HP — не числовая шкала, а лесенка из четырёх состояний: рисуют на
бумаге четыре сегмента и закрашивают/зачёркивают сверху вниз.

```
Цел → Задет → Плохо → Повержен (КО)
```

Базовый успешный удар снимает один сегмент. Дальше — качественные поправки,
решаются на словах игроками/ведущим, не подгоняются под цифру:

- **Стихия бьёт по пентаграмме (Р24)** — «сильнее»: снять дополнительный
  сегмент.
- **Стихия слаба против цели** — «слабее»: сегмент не снимать вовсе, если
  удар совсем невыгодный, или снять как обычно, если разница небольшая —
  решает ведущий по ощущению, не формулой.
- **Заклинание дорогое/крупное (по Листу 3, «Гнев бури»-уровень)** — бьёт
  заметно сильнее «Искры»: базово снимает на один сегмент больше.
- **Щит (дуга) на пути удара** — гасит один сегмент; если щит той стихии,
  что контрит атаку по пентаграмме — держит целиком, урон не проходит.
- **Вес модификатора (правило 5)** — крупно нарисованный `/` или `\`
  сдвигает эффект ещё на полшага: при спорном случае трактуйте в пользу
  того, что нарисовано крупнее.

Это протокольный приём для партии на бумаге, а не механика игры — в цифровой
версии считает формула Р4 (`точность × f(размер)`), здесь её сознательно
заменяет решение людей за столом, чтобы проверить сами правила Р26, а не
конкретные коэффициенты.

## Чек-лист наблюдений

Фиксировать по ходу партии, коротко, сразу как заметили — не восстанавливать
по памяти в конце:

- [ ] **Споры «валидно/осечка»** — какой рисунок, какое правило (1–6),
      сошлись ли на решении или нет.
- [ ] **Какой вид осечки встречался чаще всего** — А (две стихии) / Б
      (глагол без стихии) / В (глиф на границе) — и ощущался ли какой-то
      из них как «нечестный» или неожиданный для игрока.
- [ ] **Cost-модель на ощупь** — подтвердилось ли «больше орбит — дороже,
      дольше рисовать, сильнее бьёт» (Р21); был ли момент, когда маленькое
      заклинание внезапно казалось сильнее большого (или наоборот) не по
      правилам, а по ощущению несправедливости.
- [ ] **Пентаграмма в бою** — использовали ли игроки контрпик осознанно,
      или про неё забывали в процессе и подглядывали в Лист 1 каждый раз.
- [ ] **Замыкающий круг** — удобен ли как жест подтверждения; были ли
      случаи «не замкнул — обидно, каст пропал»; не путали ли круг с
      частью композиции.
- [ ] **Скучные места** — раунды/обмены без новых решений, повторение
      одного и того же заклинания без причины, паузы ожидания.
- [ ] **Что пришлось объяснять руками** — любой момент, где игрок не
      разобрался по Листам 1–3 без словесной подсказки ведущего; что
      именно осталось непонятным по шпаргалке.
- [ ] **Общее впечатление игроков** — понравилось ли, попросили ли ещё
      раунд, что назвали в пересказе своими словами.

## Критерии успеха эксперимента (Р36, эксперимент 2)

Р36 фиксирует сам факт эксперимента («рисуем и кастуем по правилам Р26»),
но не даёт числового порога успеха для него отдельно от эксперимента 1.
Ниже — критерии, выведенные из цели корпуса Р32 («на нём проверяют, что
правила Р26 однозначны, а относительные стоимости ощущаются справедливо»,
`docs/game/spellcraft.md`) и задачи issue #12:

- **Правила однозначны.** Большинство кастов на партию разбираются по
  Листу 2 без спора «валидно/осечка»; спорные случаи единичны, а не
  систематичны по одному и тому же правилу.
- **Стоимость ощущается справедливой.** Игроки на словах подтверждают
  cost-модель «больше орбит — дороже и дольше, но сильнее» (Р21), а не
  считают её обманом (см. чек-лист).
- **Пентаграмма читается в бою.** Контрпик замечают и используют, не
  игнорируют как незначащий довесок.
- **Протокол самодостаточен.** Новый игрок, не участвовавший в разработке
  игры, разбирается по Листам 1–3 без пояснений на ходу — прямой аналог
  критерия эксперимента 1 из Р36 («сам понял каст за минуту»).

Формального порога («сколько споров — уже провал») decisions.md не задаёт —
это открытый вопрос, см. ниже.

## Открытые вопросы

- Р36 не фиксирует числовой порог успеха для эксперимента 2 отдельно от
  эксперимента 1 — критерии выше выведены из Р32/issue #12, а не
  процитированы напрямую; если после партии окажется, что этого мало для
  однозначного вывода «прошли/не прошли», порог стоит закрепить в
  `decisions.md` отдельной записью.
- Лесенка HP «Цел → Задет → Плохо → Повержен» и её качественные поправки —
  протокольное изобретение этого документа, не решение из `decisions.md`;
  если партия покажет, что она даёт нечестные или нечитаемые исходы,
  меняйте её на месте, не переписывая правила Р26.
- Не зафиксировано, играет ли владелец сам одним из двух магов или ведёт
  партию со стороны как судья — протокол выше рассчитан на любой вариант,
  но роль ведущего стоит закрепить перед стартом.
