# docs — документация InkArena

Здесь живёт всё, что не код. Главные исполнители на проекте — ИИ-агенты,
поэтому структура заточена под быстрый вход в контекст: в корне — только
определяющие документы, детали — по папкам.

## Карта

```
docs/
├── README.md          ← ты здесь: карта и правила
├── goals.md           ← зачем существует репозиторий, критерии успеха
├── architecture.md    ← архитектура: принципы, структура, стек, зависимости
├── decisions.md       ← ЕДИНЫЙ журнал решений (Р1, Р2, ...): что, когда,
│                        почему; решение без записи здесь не существует
├── risks.md           ← риски: принятые и потенциальные, митигации, сигналы
├── roadmap.md         ← вехи: фаза 0 → MVP → альфа → бета
│
├── game/              ← GDD: источник правды об игре (витрина)
│   ├── vision.md      ← концепт на страницу: фантазия, игрок, уникальность
│   ├── spellcraft.md  ← ЯДРО: грамматика рисования, стихии, чернила, осечки
│   ├── combat.md      ← дуэль: темп, раунды, чтение противника (появится)
│   ├── modes.md       ← режимы: ранги, ПвЕ, турниры (появится)
│   ├── progression.md ← мета: билды, артефакты, экономика (появится)
│   ├── ux.md          ← экраны, блокнот мага, онбординг (появится)
│   └── world.md       ← сеттинг, тон, стиль, референсы (появится)
│
├── design/            ← кухня фазы 0: сырьё и история проработки идеи
│   ├── questions.md   ← рамка вопросов (волны 1–3) — все закрыты
│   ├── answers.md     ← сырые ответы владельца (пишет только он)
│   ├── braindump.md   ← свободный поток идей (пишет только владелец)
│   ├── discussion.md  ← темы Т1–Т9 — все закрыты, история разбора
│   ├── proposals.md   ← черновые решения — приняты, история
│   └── handoff.md     ← состояние работы: где остановились, что дальше
│
├── tech/              ← детальные тех-спеки (появятся после ядра game/)
│
└── prototype/         ← архив первого прототипа (код в src/ пока жив)
```

Файлы `game/` и `tech/` не создаются заранее — каждый рождается, когда в
`decisions.md` накопилось достаточно закреплённого. Папку спек и планов
суперсил (superpowers) создаёт сам плагин при первом использовании —
руками её не создавать (Р39).

## Порядок чтения для агента

Вход в контекст новой сессии: `goals.md` → `roadmap.md` → `architecture.md`
→ `game/vision.md` — этого достаточно для большинства задач. Для работы над
механиками — профильный файл `game/` и связанные Р-записи в `decisions.md`;
за историей «почему так» — `design/`.

## Правила

- **game/ — источник правды об игре.** При противоречии с design/ прав
  game/ (design — черновики и история). Код и баланс документируют сами
  себя: `src/`, `src/config.ts` — числа баланса в доках не фиксируются.
- **Каждое решение — запись в `decisions.md`:** что, когда, почему.
  Изменение закреплённого — новая запись «пересмотрено», не правка старой.
- Сырые файлы (`answers.md`, `braindump.md`) пишет только владелец;
  задним числом не правим.
- Один файл — одна тема. Разросся — делим.
- Устаревшее не удаляем молча: либо в архив (как `prototype/`), либо
  пометка в шапке «устарело, см. X».
- Решение без «почему» — не решение.
