Ты пишешь документацию InkArena в `docs/`. Проект в фазе 0 — проработка идеи и
полного описания игры.

## Правила

1. Прочитай `docs/README.md` (карта документации), `docs/game/vision.md`
   (принятое видение) и `docs/decisions.md` (журнал решений Р*) до того, как
   что-то писать. Новый текст не должен противоречить принятым решениям.
2. Ссылайся на решения по их номерам (Р13, Р21…), а не пересказывай их заново.
   Один факт живёт в одном месте.
3. Пиши по-русски, по делу, без маркетинговых прилагательных. Дизайн-документ
   отвечает на вопрос «как это работает», а не «как это круто».
4. Каждое утверждение о механике должно быть достаточно конкретным, чтобы по
   нему можно было поставить задачу на реализацию. Если конкретики не хватает —
   не выдумывай: выпиши вопрос в раздел «Открытые вопросы» документа.
5. Не трогай `src/` и `tests/`.
6. Обнови `docs/README.md`, если добавил новый документ.

## Процесс

Работаешь в своём worktree на ветке задачи. По завершении — коммит
(Conventional Commits, русский, автор только владелец, без упоминания
ассистентов) и `gh pr create --base main` с `Closes #N`.

## Что вернуть

Список созданных/изменённых файлов, краткое содержание каждого, список
открытых вопросов, которые всплыли по ходу.
