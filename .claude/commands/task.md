Возьми в работу issue: **$ARGUMENTS**

Для КАЖДОГО указанного номера:

1. Прочитай issue целиком: `gh issue view <N>`. Если на issue висит метка
   `needs-decision` или `blocked` — в работу НЕ бери: сначала реши вопрос
   с владельцем / дождись блокера, потом возвращайся.
2. Определи исполнителя по метке `type:`
   - `type:design` или документация → агент `gdd-writer`
   - всё остальное, что меняет `src/` или `tests/` → агент `impl`
   Модель бери из раздела «Модель» в теле issue; если его нет — `opus` для
   `size:L` и `size:M`, `sonnet` для `size:S`.
3. Спауни агента через Agent tool с `isolation: "worktree"` — он получит свой
   изолированный git worktree и не столкнётся с другими агентами.
   В промпте передай: номер issue, полный текст issue, имя ветки
   (`<type>/<краткий-слаг>-<N>`) и напоминание про `Closes #N` в PR.
4. Если номеров несколько — спауни всех агентов **одним сообщением**, чтобы они
   работали параллельно.

Пока агенты работают, ничего не пиши в те же файлы сам.

Когда все вернутся — выведи сводку: issue → PR → статус тестов → что требует
моего решения. Не мержи ничего без моей команды (`/land`).
