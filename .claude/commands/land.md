---
description: Отревьюить PR и, если чисто, замержить с полной уборкой
argument-hint: <номер PR> [ещё номера]
---

Проведи PR через ревью и мерж: **$ARGUMENTS**

Текущие PR: !`gh pr list --json number,title,headRefName,isDraft`

Для каждого номера:

1. Спауни агента `reviewer` (модель opus) с номером PR. Если PR несколько —
   всех одним сообщением, параллельно.
2. По вердикту ревьюера:
   - **МЁРЖИТЬ** → `gh pr merge <N> --squash --delete-branch`
   - **ДОРАБОТАТЬ** → спауни `impl` в том же worktree/ветке с промптом,
     который вернул ревьюер. Потом ревью заново.
   - **ПЕРЕДЕЛАТЬ** → не мержи, покажи мне разбор и спроси, что делать.
3. После успешного мержа обязательно убери за собой:
   - `git worktree remove .claude/worktrees/<ветка>`
   - `gh issue close <N> -c "<итог: что сделано, ссылка на PR>"`
   - убедись, что локальная ветка удалена (`git branch -d <ветка>`)
4. `git worktree list` — покажи мне, что осиротевших worktree не осталось.

В конце: сводка «замержено / на доработке / отклонено» и что осталось в бэклоге.
